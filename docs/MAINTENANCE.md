# Documentation Maintenance Checklist

Use this checklist when updating or releasing.

- [ ] Update `docs/api-reference.md` when MCP tools or REST endpoints change.
- [ ] Update `docs/reference/environment-variables*.md` when config defaults or env vars change.
- [ ] Update `docs/getting-started.md` if install/run flows change.
- [ ] Update `docs/security.md` if auth/permission behavior changes.
- [ ] Review `docs/README.md` index for new/removed docs.
- [ ] Validate examples against the current CLI and tests.
